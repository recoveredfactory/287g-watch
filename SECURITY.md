# Security Policy

## Reporting a vulnerability

If you discover a security issue in 287(g) Explorer — the website, the data
pipeline, or the deploy infrastructure — please report it privately rather than
opening a public issue.

Email **davideads@grupovisual.org** with a description of the issue and, if
possible, steps to reproduce. We'll acknowledge your report and work with you on
a fix and disclosure timeline.

Please **do not** use this channel for data corrections or accuracy questions —
those are not security issues. Open a regular
[issue](https://github.com/recoveredfactory/287g-explorer/issues) for those, and
see the [methodology page](https://287g.recoveredfactory.net/methodology) for how
the data is collected.

## Scope

In scope: the SvelteKit app, the ingestion pipeline, and the SST/AWS deploy
configuration in this repository. The underlying 287(g) data originates upstream
from [appelson/Tracking_287g](https://github.com/appelson/Tracking_287g) and ICE's
published filings.
